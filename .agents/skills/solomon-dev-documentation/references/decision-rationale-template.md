# Decision Rationale Template

Use this block in module docs for important decisions.

## Decision
- One-sentence description of chosen approach.

## Context
- What constraints and risks made this decision necessary.

## Alternatives Considered
- Alternative 1 and rejection reason.
- Alternative 2 and rejection reason.

## Tradeoffs
- What this improves.
- What complexity or limits it introduces.

## Consequences
- Impact on module boundaries, contracts, tests, and operations.
- Future extraction or migration implications.

## Evidence
- Pointers to relevant files and canonical spec references.
